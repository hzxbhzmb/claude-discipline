# 多会话并发纪律

> 同项目可能起多个 Claude Code 会话同时工作，共用 `todo/current.md` / `research/` / `methodology/`。本文件描述并发安全机制。

## 任务段归属会话

任务段标题**必须带本会话 sessionId 短 ID 标注**：

```markdown
## YYYY-MM-DD — 任务简述 <!-- session: xxxxxxxx -->
```

SessionStart 时插件在规则尾部告诉你本会话短 ID，照抄即可。

**Hook 只认带本会话标注的段**：握手检查、验算检查都按本会话过滤；别的会话和祖传无标注段对你"透明"（不拦截、不连坐）。

**没带标注 = 没建任务段**：`pre-edit-write` hook 会拒绝你对项目文件的编辑，提示把段标题补上 `<!-- session: xxxxxxxx -->`。

## 禁止 Write 整覆盖 todo/current.md

会吞掉并发会话正在写入的段。**必须用 Edit 增量改动**——Edit 的 `old_string` 精确匹配是天然的乐观锁。

如需"重写"：
1. 先用 Edit 清空你自己会话标注的段
2. 再用 Edit 追加新内容
3. 绝不要整文件 Write

## 证据日志按会话隔离

`/tmp/claude-evidence-${sessionId}.jsonl` 每会话独立。SessionStart 只清自己会话的或 >24h 陈旧的，绝不无差别全删。

## 升级零摩擦：自动认领祖传段

老版本（不支持多会话）建的任务段没带 session 标注。SessionStart 自动把"最近一个进行中的祖传段"（非归档、无标注、含 `[ ]`）贴上本会话标注，B 类用户（有在做任务）升级零动作。

**认领规则**：
- 已归档段：不认领
- 已带任何 session 标注：不认领（可能是别会话用着）
- 纯空段（无 `[ ]` 也无 `[x]`）：不认领
- 全部 `[x]` 已完成：不认领（不是"在做"的任务）

**多个候选时**：取文件里最后一个（最近建的）。其它仍是祖传段，对新 hook 透明。

## 与"祖传段"的关系

"祖传段" = 标题没带 `<!-- session: xxxxxxxx -->` 标注的任务段。**所有 hook 对祖传段一律透明**——不拦截、不连坐。所以升级时不需要回头改历史段。
