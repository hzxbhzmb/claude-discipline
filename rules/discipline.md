# Claude Discipline 纪律规则

> 本规则由 claude-discipline 插件自动注入，通过 Hook 系统级强制执行。

## 工作流总览

```
三次握手 → 执行 → 四次挥手
  ↓         ↓        ↓
对齐意图   证据链    验算验收
```

1. **读 todo/current.md** → 完成三次握手（理解 → 确认 → 授权）
2. **读 methodology/_index.md** → 按场景查阅方法论
3. 逐个执行子任务 → **有工具调用** → 标记 `[x]`
4. 全部完成 → **用工具验算** → 写 `> ✅ 验算通过` → 向用户汇报 → 等用户验收

## 三次握手（Hook 强制）

**"收到任务就开干"被禁止。** Hook 检查当前任务段是否有 `> ✅ 执行授权`，没有 → 系统级阻断。

| 握手 | 做什么 |
|------|--------|
| **SYN** — 用户发任务 | 在 todo 新建任务段，记录 `**用户意图**` |
| **SYN-ACK** — AI 回传理解 | 写 `**AI 理解**`（目标/边界/路径/风险），写 `> 🤝 待用户确认`，在对话中向用户确认 |
| **ACK** — 用户确认 | 用户确认后写 `> ✅ 执行授权：{确认要点}`。用户要求修正 → 更新理解，重走 SYN-ACK |

握手完成后写 `**达标标准**` 和子任务列表，然后才能编辑项目文件。

### 任务段完整格式

```markdown
## YYYY-MM-DD — 任务简述

**用户意图**：（忠实记录，不加工）

**AI 理解**：
- 目标：（用你的话重述）
- 边界：（什么不做）
- 路径：（关键步骤）
- 风险：（最担心什么）

> 🤝 待用户确认
> ✅ 执行授权：用户确认方向正确

**达标标准**：...

- [ ] 子任务 1
- [ ] 子任务 2
```

## 证据链（Hook 强制）

每次工具调用自动记录到会话日志（工具名、目标、时间戳）。AI 无法伪造。

| 操作 | Hook 检查 | 不通过 |
|------|----------|--------|
| 标记 `[x]` | 自上次标记以来有非 todo 的工具调用？ | **拒绝** |
| 写 `✅ 验算通过` | 自最后一个 `[x]` 以来有读取/执行类工具调用？ | **拒绝** |

有效证据：Read / Edit / Write / Bash / Grep / Glob / MCP 工具调用。
不算证据：仅编辑 todo/current.md 或 methodology/_index.md。

## 四次挥手（关闭协议）

| 步骤 | 做什么 |
|------|--------|
| AI 验算 | 全部 `[x]` 后用工具验算，写 `> ✅ 验算通过：{方法和结果}` |
| AI 汇报 | 在对话中告知用户：做了什么、验算了什么、已知局限 |
| 用户验收 | 确认关闭 / 要求补充 / 发现方向偏了则重新握手 |
| 正式关闭 | 如有方法论沉淀，写入 methodology/ |

### 验算举例
- 改了代码 → 真的跑一下
- 修了 bug → 构造原始触发条件验证
- 下载了文件 → ls 确认存在且大小合理
- 转换了格式 → 读输出文件确认内容完整

## 任务管理补充规则

- 尽量使用多 agent 并行执行独立的任务项
- **逐个完成、逐个标记 `[x]`**，不要攒到最后批量标记
- 归档：todo/current.md 超 80 行时，将已完成章节按月归档到 `todo/archive/YYYY-MM.md`

## 研究文档（Hook 强制）

- 研究/分析/调研的产出**必须写入 `research/` 目录**，不可只在对话中输出
- 拆子任务时，涉及研究的子任务**必须显式包含 `research/` 路径**（如"分析 XX → 写入 research/"）
- **Hook 检查**：标记 `[x]` 时，若子任务文本含 `research/`，证据日志中必须有对 `research/` 目录的写入操作，否则拒绝
- 文档用 Markdown 格式、中文撰写，专有术语可保留英文

## 方法论

方法论存放在 `methodology/` 目录，顶层索引 `_index.md`。

- **查阅**：任务开始读 `_index.md`，按场景只读匹配的详情文件
- **写入**：反思抽象为方法论，写入对应分类；写入前先查索引避免重复
- **整理**：某分类超 5 个文件时加二级索引；定期合并重叠条目

## 严禁的反模式

- ❌ 跳过三次握手直接开干
- ❌ 回复"收到"就开始做，没有回声式复述理解
- ❌ 做完再补 todo（事后补账）
- ❌ 批量标记 `[x]`（攒到最后一起标）
- ❌ 自己说 done 就结束，不等用户验收
